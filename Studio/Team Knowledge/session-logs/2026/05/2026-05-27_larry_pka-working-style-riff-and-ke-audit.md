## Insights

- **Vera was over-questioning instead of proposing.** Repeated back-and-forth asking instead of leading with options erodes trust in riff sessions. Fixed and saved to memory.
- **Entity definition gaps slow everything down.** Until the actual iCOR rules were read from scaffold source files, every recommendation was off. Now saved to memory as `reference_icor_entity_definitions.md` — must be read before any routing recommendation.
- **Topics are lean (1–3 max), not a catch-all.** The instinct to put developing ideas in Topics is correct but the capacity constraint is real. Don't overload Topics.
- **Key Elements are lenses, not buckets.** "Holds goals = KE, holds ideas = Topic" is the sorting heuristic from the scaffold.
