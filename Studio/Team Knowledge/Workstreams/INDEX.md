# Workstreams - Index

**Workstreams are multi-agent compositions.** A Workstream describes how more than one specialist, often plus Alyssa, collaborates to deliver a recurring outcome. Where an SOP is a single-agent skill, a Workstream is the choreography that strings skills together.

Workstreams reference SOPs and Guidelines via `[[wikilinks]]`. They never duplicate the steps or rules those files contain.

Filename pattern: `WS-NNN-<title>.md`. See [[GL-001-file-naming-conventions]] for slug rules.

## When to write a new Workstream

- More than one specialist is involved.
- The activity recurs on a schedule or recurring trigger.
- The choreography matters as much as the steps.

If only one specialist is involved, write an SOP instead. If the rule is static and never executed, write a Guideline instead.
