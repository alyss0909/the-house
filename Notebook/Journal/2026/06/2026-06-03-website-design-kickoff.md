# Website Design Kickoff — Editor in Residence

Three-way call with [[gillian]] and [[andrew]] to kick off the website build for [[gillian]]'s brand, Editor in Residence. Call started around 1:30 PM, ran about 19 minutes. Andrew is a new contact — Squarespace designer brought in to build the site.

## What we covered

The main agenda was to sync on where things stand, assign tracks of work, and unblock everyone so parallel progress can happen.

**Homepage first.** Andrew's process is always to lock the homepage before touching other pages. The homepage establishes the design language — fonts, layout rhythm, visual weight — and everything else follows from it. Once we're happy with homepage, he builds out the rest.

**Font direction.** The aesthetic is magazine-classy: serif headings, sans-serif body. References on the table included The Impression and photographics magazine. Andrew will mock up 4-5 font combinations delivered as a Figma file or PDF so we can click between options and work down the tree. He noted that knowing what we hate is as useful as knowing what we love.

**Logo direction.** Moving away from the blocky Jacques Marie Mage style. Jillian is drawn to black/white/gray with a graphic treatment on "Editor in Residence" — possibly outlined lettering or text that steps back visually. Andrew will explore options. No hyphens in "Editor in Residence" — confirmed by the group.

**Text over images.** Andrew flagged this clearly: do not bake text into images. The font treatment should be implemented as a CSS overlay. This keeps the images clean, protects SEO, and makes future changes possible without re-exporting everything. Alyssa confirmed she would not be making images with text baked in.

**FlowDesk.** Jillian is paying for it but not actively using it yet. Will be used for newsletter signup and lead capture when the site is ready. Andrew is happy to wire in whatever embed codes are needed; that's on Jillian and Alyssa to set up.

**IDX embed.** Alyssa has done preliminary research on IDX providers for real estate listings. Andrew confirmed the embed will be an iframe — a JavaScript embed code that loads the third-party service. Squarespace can accept it without issues. The key evaluation criteria for picking a provider are display quality and responsiveness (resize the browser and see what happens), plus any custom styling options the provider offers. Andrew offered to test an embed once a provider is shortlisted, or even before the site build starts if there's a trial available.

**Andrew's process recap.** Homepage rounds first, then scope/agreement, then remaining pages. Scope of work and budget conversation will happen separately between Jillian and Andrew.

## Decisions made

- Homepage is locked as the starting point before any other page work begins.
- Font explorations: Andrew delivers Figma/PDF with 4-5 combinations; iterative rounds until decided.
- Logo: open exploration in black/white/gray; graphic treatment for "Editor in Residence" text; no hyphens; moving away from blocky sans-serif.
- Text over images must be CSS overlays, not baked into image files.
- IDX evaluation criteria: display quality, responsiveness, and custom styling options.
- FlowDesk will be integrated later; no action needed now.
