# Fable Content Relay Continuation Handoff

## Read This First

Alyssa is not asking for a generic AI content generator.

The actual project is to use Fable 5 once, while she has access, to deeply understand her real Second Brain and use that pass to build a local, source-grounded content relay. The relay should turn Alyssa's real ideas, emails, Obsidian notes, BOH modules, weekly themes, post types, best performers, and exact CTA map into usable weekly content batches without her re-explaining her strategy every time.

Success means it feels like Alyssa's strategy and taste are finally operating as a low-token weekly machine that retrieves, matches, deepens, scores, and exports.

Failure means it becomes another generic AI content generator that invents strategy, flattens nuance, ignores source truth, writes for beginners, or produces "technically fine" content Alyssa still has to wrestle into being.

## Alyssa's Top-Line Intention

Build a team of agents that can eventually run the content production system:

- Based on Alyssa's best-performing posts.
- Able to repurpose emails.
- Following weekly themes/focuses.
- Generating hooks, captions, and first comments.
- Building concept arcs using copy frameworks such as AIDA when useful.
- Producing slide breakdowns: hook page, second page, teaching pages, CTA page.
- Adding SEO / keywords where relevant.
- Finding the matching freebie and/or product.
- Adding the correct CTA word / ManyChat word from the canonical CTA map.
- Running as an agentic relay, not one flat prompt.
- Turning one idea into edu, offer, and For The Girls versions.
- Then adapting the same idea for email, Pinterest, productivity, and systems contexts.
- Possibly requiring Alyssa to "gold standard" one of each format first.
- Turning emails into carousels.
- Matching concepts to the most relevant BOH module and using that module to teach one small thing in a nuanced way.
- Creating an intentional gap that makes someone want a freebie, product, or offer.
- Delivering final carousel copy into Notion and/or Canva.
- Eventually tagging the designer when ready for design.

## Fable 5 Role

Alyssa has Fable 5 access only for a short window. The point of using Fable 5 is not to build the final production system inside Fable forever.

The point is:

- Use Fable's large context to deeply understand Alyssa from her Second Brain.
- Create a usable "Alyssa clone" layer: voice, strategy, decision logic, teaching style, taste, and offer logic.
- Use that understanding to help tune or instruct a local Hermes/OpenClaw-style agent later.
- Then build a content relay that can use the Alyssa clone to create weekly content stacks.

The handoff should prepare Fable to ingest the right sources in the right order, with the right cautions.

## Hard Source Rules

This project lives or dies by source truth.

Fable / the relay should:

- Treat Alyssa-original writing as highest trust.
- Treat analysis files as interpretation, not direct source truth.
- Label AI-derived materials clearly.
- Avoid training on random AI outputs as if they were Alyssa.
- Never invent offers, CTAs, themes, ManyChat words, or product links.
- Never write for total beginners unless a source or brief explicitly says that is the target.
- Preserve Alyssa's nuance, taste, and audience sophistication.

## Suggested Agent Relay Shape

This is a proposed shape, not yet implemented:

1. Source Retriever
   Finds the right Alyssa-original notes, BOH lessons, emails, carousels, and product facts.

2. Strategy Matcher
   Decides which weekly focus/theme, post type, product/freebie, and CTA lane fit.

3. BOH Teaching Extractor
   Pulls one small, nuanced teaching point from the most relevant BOH module.

4. Concept Architect
   Builds the concept arc, possibly with AIDA or another copy framework when useful.

5. Wren / Alyssa Voice Writer
   Writes hooks, slides, captions, first comments, emails, Pinterest descriptions, etc. in Alyssa's voice.

6. CTA Mapper
   Checks product/freebie/offer fit and exact ManyChat word against Notion.

7. Quality Scorer
   Scores source truth, nuance, voice, offer fit, usefulness, and non-genericness.

8. Exporter
   Formats the final batch for Notion, Canva, and designer handoff.

8. Designer handoff spec
   Need a final output format that a designer or Canva workflow can actually use.
