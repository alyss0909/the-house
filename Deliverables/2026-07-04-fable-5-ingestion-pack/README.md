# Fable 5 Ingestion Pack

This folder prepares Fable 5 to understand Alyssa's source-grounded content system before any relay is built.

The goal is not to make a generic content generator. The goal is to help Fable build a grounded "Alyssa clone" layer that understands voice, strategy, teaching logic, taste, offer logic, and CTA discipline well enough to later support a local content relay.

## How to use this pack

Use [[07-fable-paste-prompt]] as the actual prompt. Keep it lean and outcome-focused.

The other files are context Fable can use if helpful. They are not a required procedure, and Fable should use her own judgment about what to inspect, compare, and push back on.

## Honesty frame

Where any note says something is built, ready, locked, canonical, complete, or source-truth, treat that as a claim to verify against the surrounding files.

Assume less is built than it looks. Push back if the relay idea depends on a missing map, stale CTA, unverified product fact, or AI-derived example.

## Access / connectors likely needed

Fable can work from uploaded files for the vault and Second Brain.

Fable needs Notion access, a Notion connector, or a current export/screenshot of the Products DB if Alyssa wants exact product facts, offer statuses, links, prices, and chat words verified. Without Notion access, Fable should mark product/CTA facts as unverified and never invent chat words.

No Canva connector is needed for this ingestion pass. Canva or design access matters later, once the relay is ready to export designer handoffs.

## Current boundary

Do not build the full relay yet.

The first useful Fable output is a source-grounded ingestion assessment and clone brief:

- What Fable now understands about Alyssa's voice, strategy, teaching style, audience, taste, offers, and CTA logic.
- What source files it relied on.
- What it refused to learn from because the source was interpretive, AI-derived, stale, or low-trust.
- What gaps must be closed before a local relay can reliably produce edu, offer, and For The Girls batches.
- A proposed local relay shape that can be implemented later.
