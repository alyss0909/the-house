# Source Trust Manifest

Fable should learn Alyssa from source truth first, interpretation second, and AI-derived material only as labeled scaffolding.

The core failure mode is a model that writes "technically fine" content that ignores Alyssa's real source material, flattens nuance, invents CTAs, or writes for generic beginners.

## Highest-trust source truth

Use Alyssa-original work as the strongest evidence: real pages, real emails, real captions, real carousel slides/screenshots, BOH program materials, transcripts, sales copy, journal notes, and offer notes.

The Second Brain is the main source corpus. The broader vault can be used for context and idea history.

## Interpretation, not raw voice

Analysis files, frameworks, templates, routing guides, and Wren memory are useful, but they are interpretation. They should help Fable notice patterns and navigate faster; they should not outweigh the real source examples.

## Low-trust or excluded for clone training

Do not train the clone layer on AI-derived drafts, stale brief banks, templates, or generated voice profiles as if they are Alyssa's real voice.

Known caution: `PKM/Second Brain/examples/soft-sundays/me-i-need-ideas-i-already-had.md` was explicitly flagged by Alyssa as too AI.

## Wider vault rule

Fable may inspect the entire vault for context, but it must report which files it used and what trust tier each file belonged to.

Do not silently blend raw Alyssa examples, analysis notes, generated templates, and stale drafts into one undifferentiated model of "Alyssa."

## Product and CTA rule

For offer meaning, read `PKM/My Life/Offers/`.

For current product facts, links, prices, statuses, themes, and chat words, use the Notion Products DB described in [[04-product-cta-bridge]].

Never invent a ManyChat/chat word. If a product has no chat word in Notion, mark it as `manual selection needed`.
