# Alyssa Clone Key

## Objective

Use Fable 5 once, while Alyssa has access, to deeply understand Alyssa's real Second Brain and create a source-grounded clone layer for a future local content relay.

This clone layer should understand:

- Alyssa's writing voice.
- Alyssa's strategic taste.
- Alyssa's teaching logic.
- Alyssa's offer and CTA judgment.
- Alyssa's audience sophistication.
- Alyssa's content rhythm across edu, offer, and For The Girls slots.

## Success

This works when Fable can explain Alyssa's content system from evidence and help prepare a relay that can produce weekly content batches without Alyssa re-explaining her strategy every time.

The eventual local relay should retrieve source truth, match the idea to the right slot and offer, deepen it with BOH teaching where useful, score it against Alyssa's standards, and export usable carousel/caption/CTA materials.

## Failure

This fails if Fable produces a generic AI content generator.

Red flags:

- It writes for total beginners by default.
- It invents offers, CTAs, chat words, product links, or themes.
- It treats analysis files as if they are Alyssa-original examples.
- It flattens Alyssa's nuance into broad marketing advice.
- It makes every post sound like a sales post.
- It treats Friday For The Girls posts as disposable personality content.
- It ignores BOH modules when the idea needs a stronger teaching point.
- It outputs content Alyssa still has to wrestle into being.

## What is already known

Alyssa's content relay needs two layers:

- A deep Alyssa clone trained on source truth.
- A production relay that uses that clone to retrieve, match, deepen, score, and export content.

The first build target should stay modest: one source-grounded weekly content stack before attempting the full relay.

## What Fable should learn

From Alyssa-original sources:

- What her strongest real voice sounds like across carousels, emails, pages, BOH lessons, and captions.
- How she makes abstract strategy concrete through food, restaurant, domestic, calendar, and sensory metaphors.
- How she teaches one specific useful thing without becoming basic.
- How she creates an intentional gap toward a freebie, product, or offer.
- How she uses identity and permission in For The Girls content.
- How her email-first, Pinterest, BOH systems, slow marketing, and productivity lanes connect.

From interpretation sources:

- Which patterns past analysis has extracted.
- Which patterns are useful enough to become relay rules.
- Which analysis claims need verification against raw examples.

From product/CTA sources:

- Which offers belong to each theme.
- Which offers are freebies vs low-ticket vs flagship.
- Which exact chat word belongs to the offer, if Notion has one.
- Which offers require manual CTA selection because Notion has no chat word.

## Honest state

Strong source material exists, especially in the Second Brain, BOH program files, carousel examples, sales pages, and offer notes.

The missing pieces are still real:

- A complete BOH module-to-public-content map.
- Best performer cards for each carousel.
- A durable local product/CTA bridge that references Notion without duplicating volatile facts.
- A clear email-to-carousel transformation rule.
- A designer/Canva handoff spec.
- A hard quality rubric for source-grounded vs generic output.

Fable should help identify these gaps and propose the smallest next build step.
